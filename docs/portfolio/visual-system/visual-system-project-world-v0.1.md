# Visual-system project world v0.1

## Purpose and hierarchy

Visual-system pages explain premise, selected specimens, rules, states, accessibility, implementation evidence, versioning, and limitations. The production order moves process before evidence so authored decisions frame what each specimen proves.

The standard shell and specimen-led layout use an editorial measure, balanced metadata, inspectable evidence, and a wider specimen stage. Specimens remain selected explanations rather than exhaustive token dumps.

## Media, proof, and fallbacks

The lead preference is an explanatory specimen or a type-led stage. Functional glyphs and state specimens remain mark-free. Contrast and state proof must include text or shape reinforcement. No component or primitive color is invented.

At narrow widths and zoom, specimen, process, and evidence sequences collapse to the normal document flow. Forced colors removes decorative surfaces, print retains explanatory content, and no JavaScript is required. Authored marks are Level 0. Approval checks include the visual-system fixture, headings, specimen alternatives, visible limitations, and semantic state proof.
