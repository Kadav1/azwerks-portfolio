# Software project world v0.1

## Purpose and hierarchy

Software pages explain the concrete tool, its purpose and workflow, interface evidence, maintenance, releases, limitations, and source availability. The production order is orientation, product header, interface/type-led stage, narrative, evidence, limitations, process, releases, links, provenance, relations, and context navigation.

The technical shell and technical layout keep state and release material inspectable. Narrative uses the technical reading measure; metadata is compact and evidence inspectable. Interface media is evidence and is never placed behind fake device chrome.

## Media, proof, and fallbacks

Real interface media is preferred; absence uses the shared type-led no-media state. Evidence and limitations remain adjacent in the reading sequence. Releases are emphasized without implying that the first item is current. Private source availability is text only and never exposes a path.

The world adds no CTA band, SaaS metrics, uptime, users, stars, downloads, dashboard theater, terminal styling, glow, or autoplay. It retains one column under pressure, local code/table overflow, native links, one H1, 44px shell controls, forced-color meaning, print content, and no-JavaScript access. Authored marks are Level 0. Rich and sparse software fixtures are required approval checks.
