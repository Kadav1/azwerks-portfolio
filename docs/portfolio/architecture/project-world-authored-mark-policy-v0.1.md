# Project world authored-mark policy v0.1

Allowed authored-mark levels are 0 and 1. This release activates Level 0 for all five worlds because no approved mark asset or approved non-functional surface exists.

Level 1 would require an authorship, inspection, revision, ownership, provenance, or workshop purpose; an approved removable surface; no overlap with content or controls; clean and reduced fallbacks; forced-colors removal; print removal unless documentary; no state meaning; no uncontrolled color; and a successful removal test.

Marks are prohibited on artwork, captions, rights, provenance, code, architecture, evidence, states, metadata, functional glyphs, navigation, actions, and controls. CSS pseudo-elements do not invent graffiti-like decoration. This phase adds no mark assets.
