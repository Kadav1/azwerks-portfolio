# Project detail world extension contract `v0.1`

Date: 2026-07-15

## Stable seam

Every detail article exposes `data-project-category`, `data-project-layout-profile`, `data-project-theme-profile`, and `data-project-motion-profile`. The global shell receives only bounded existing shell/world values. Project-world aliases control canvas, surface emphasis, media surround, density, measure, type emphasis, and mark intensity; they cannot redefine functional semantics.

## Future changes allowed

The project-world phase may change composition, proportion, typography emphasis, media sequencing, local surface emphasis, authored-mark allowance, density, and chapter rhythm. Those decisions must remain content-led, token-governed, semantically linear, responsive, and static-first.

## Invariants that cannot change

Worlds cannot alter:

- `/work/<slug>/` identity or route eligibility;
- the public-safe data boundary;
- one H1, title, summary, and truthful state meanings;
- Back to Work and deterministic context navigation;
- global focus, theme, reduced-motion, target, and navigation semantics;
- native Markdown access and no-JavaScript content;
- evidence trust, metric context, visible limitations, or provenance truth;
- media rights/source/transcript rules;
- public/routable relation filtering;
- empty-section and no-media behavior.

Final category DOM orders, custom galleries, custom type systems, category decoration, scene engines, View Transitions, spatial continuity, and route choreography are explicitly deferred. A world extends the invariant; it never replaces or forks it.
