# Project world section policy v0.1

Each world owns a frozen policy specifying ordered sections, required invariant sections, narrative measure, metadata/evidence density, lead preference, and treatments for process, releases, limitations, provenance, and relations.

Orientation and header always lead. Context navigation always closes. Limitations and provenance are required invariant capabilities and cannot be configured as hidden. Empty optional sections remain suppressed by `WorldSection.astro`; policy never fabricates companion content.

The five order arrays are deliberately distinct and validated against the corresponding Astro composition. Section order may express reading need, but all content rendering remains centralized. Evidence references continue to be filtered by the invariant. Relations remain public and routable. Previous/next order remains the Work Register order.

Policy changes require registry tests, static composition validation, all five browser fixtures, narrow/zoom/print/forced-color checks, and manifest regeneration. A policy cannot remove orientation, header, limitations, provenance, or context navigation.
